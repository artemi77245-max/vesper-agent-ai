# Vesper — кодинг-агент в терминале

Маленький, но настоящий кодинг-агент на Python — в духе Claude Code / Codex.
Читает и пишет файлы, ищет по коду, запускает команды и крутит цикл
tool-calling, пока задача не решена. Работает на любой OpenAI-совместимой
модели через OpenRouter (в т.ч. бесплатные).

## Как устроен (архитектура)

Агент — это не магия, а простой цикл:

1. Отправляем историю сообщений + список инструментов модели.
2. Модель либо просит инструмент, либо отвечает текстом.
3. Если просит инструмент — выполняем его, результат кладём обратно в историю
   и повторяем. Если ответила текстом — задача решена.

Файлы:
- `agent.py` — ядро: цикл tool-calling, подключение к OpenRouter, смена модели
- `tools.py` — инструменты (руки агента) + защита опасных команд
- `session.py` — сохранение истории диалога между запусками
- `mascot.py` — ASCII-маскот (лисёнок)
- `main.py` — точка входа: приветствие, команды, чат-цикл

## Инструменты агента

- `read_file` — прочитать файл
- `write_file` — записать/перезаписать файл
- `list_files` — список файлов и папок
- `make_dir` — создать папку
- `search_code` — поиск текста по всем файлам (рекурсивно)
- `run_command` — выполнить shell-команду (опасные — с подтверждением)

## Команды в терминале

- `/model <имя>` — сменить модель на лету
- `/model` — показать текущую модель
- `/reset` — очистить историю диалога
- `/help` — справка
- `exit` — выйти

## Запуск

1. Скопируй `.env.example` в `.env` и впиши ключ OpenRouter
   (бесплатный: https://openrouter.ai/keys):
   ```
   OPENROUTER_API_KEY=твой_ключ
   VESPER_MODEL=openai/gpt-oss-20b:free
   ```
2. Установи зависимости и запусти:
   ```bash
   pip install -r requirements.txt
   python main.py
   ```
3. Пиши задачи, например: «создай файл hello.py, который печатает привет,
   и запусти его». Vesper сам прочитает/запишет файлы и проверит результат.

## Прокси (необязательно)

Если OpenRouter недоступен напрямую, укажи прокси в `.env`:
```
PROXY_URL=socks5://127.0.0.1:2080
```
Для socks5 нужен пакет `httpx[socks]` (уже в requirements.txt).

## Безопасность

- Файловые операции ограничены рабочей папкой (`workspace/`) — агент не вылезет
  за её пределы.
- Опасные команды (`rm`, `dd`, `sudo`, `git push` и др.) требуют явного
  подтверждения перед выполнением.
- `.env` с ключом в `.gitignore` — не попадёт в репозиторий.
